# Demo 视频拍摄脚本
## 参谋 · Counsel AI · 2 分钟产品 Demo
**总时长：120s | 格式：录屏 + 旁白 | 配乐：低沉 ambient，不盖旁白**

---

## 拍摄准备清单

### 技术准备
- [ ] 后端 API 确认跑通（蒋鲤）
- [ ] app-mockup-v7.html 在浏览器中打开，全屏（Command + Shift + F）
- [ ] 浏览器窗口 1080p 或 4K，隐藏标签栏
- [ ] QuickTime → 新建屏幕录制，选择浏览器窗口区域（不含 menubar）
- [ ] 录音：外接麦克风（如有），否则 AirPods 麦克风
- [ ] 提前准备好"预设困惑"文本（见下方），不要现场打字，用复制粘贴

### 内容准备
- [ ] 预设困惑文本（见 Shot 3）
- [ ] 确认 12 人并行 streaming 效果稳定（最重要的一帧）
- [ ] 提前跑一遍完整流程，知道每步大约需要多长时间

---

## 镜头脚本

> 格式：`[时间] [镜头/操作] | 旁白文字（斜体）`
> 旁白可后期配音，也可边操作边讲

---

### INTRO（0:00-0:15）— 不开屏幕录制，黑屏字幕

**画面：黑底，白字逐字出现**

```
你上次做一个真正重大的决策
你找谁商量？
```

*旁白（沉稳，慢速）：*
> "你上次做一个真正重大的决策——
> 换方向、押注、结束一段关系——
> 你找谁商量？"

---

### Shot 1：痛点闪现（0:15-0:25）— 黑屏，文字叠加

**画面：3 行字，依次出现，每行 3s**

```
你找的人，是他人生路径的囚徒。
```
↓
```
就算你问了 AI，它同意你。
```
↓
```
参谋不一样。
```

*旁白：*
> "你找的人，是他人生路径的囚徒。
> 就算你问了 AI，它只会同意你。
> 参谋不一样。"

**[制作说明]** 这 3 行用 Final Cut 或 CapCut 做字幕叠黑屏即可，不需要产品界面。

---

### Shot 2：产品界面出现（0:25-0:35）— 切到屏幕录制

**操作：** 打开 app-mockup-v7.html，停留在首页/输入页

**画面：** 产品 UI 全屏，圆桌或星环界面，12 个幕僚头像安静显示

*旁白：*
> "12 位顾问，同时在场。
> 乔布斯、老子、毛泽东、爱因斯坦——
> 他们的哲学框架，全部对你的问题展开。"

**[拍摄要点]** 停留 10s，让评审看清楚有哪些人。可以缓慢 hover 过几个头像。

---

### Shot 3：输入困惑（0:35-0:50）— 屏幕录制继续

**操作：** 点击输入框，粘贴预设困惑文本（不要打字，直接粘贴）

**预设困惑（复制备用）：**
```
我现在面临一个重大决定：是继续在现在的公司做高管，
还是出来全职做自己的产品？
我对现在的工作有积累，但总觉得这不是我真正想做的事。
最大的顾虑是：孩子刚两岁，家庭压力很大，
但如果现在不跳，未来可能更难。
```

*旁白：*
> "用户输入一个真实的困惑。
> 不用写完美——直接说就行。"

**[拍摄要点]** 粘贴速度要稳，文字出现后停 2s，再点提交。

---

### Shot 4：Facilitator 定义问题（0:50-1:05）— 屏幕录制

**操作：** Facilitator Agent 返回 2-3 个澄清问题，展示对话界面

**画面：** Facilitator 的问题出现（streaming 效果）

*旁白：*
> "Facilitator 先帮你把模糊困惑，
> 转化成可以辩论的具体问题。
> 这一步，大多数人从来没有做过。"

**[拍摄要点]** 如果 streaming 太快，可以预先录好这段回放。关键是让"锁定问题"这个概念清晰出现。

---

### Shot 5：12 人并行发言（1:05-1:30）— ⭐ 最重要的 25s

**操作：** 触发 Step 4，12 个 sub-agent 并行调用

**画面：** 12 张卡片同时出现，streaming 流出内容——每张卡片都在动

*旁白：*
> "12 个 Claude sub-agent，同时运行，互不可见。
>
> 乔布斯在问：你砍掉了什么？
> 老子在问：你是不是在逆流而上？
> 毛泽东在找：主要矛盾在哪里？
>
> 这不是角色扮演。
> 是真正的多智能体并行编排。"

**[拍摄要点]**
- 这帧是整个视频最重要的画面，必须清晰流畅
- 如果 12 个 streaming 同时出现太震撼，让它停留足够长（10-15s）
- 镜头不要乱动，让画面本身说话
- 如果 API 响应有延迟，可以用预录好的这段播放

---

### Shot 6：Harvest Round — 幕僚评价你（1:30-1:50）— ⭐ 情绪高潮

**操作：** 跳到 Step 8，展示 Harvest Round Part B 界面

**画面：** 3-5 个幕僚对用户的评价，每条一句话，直接、锋利

**预设几条示例评价（如需 mock）：**

> 乔布斯：你描述这个决定的方式，说明你还不清楚自己真正要的是什么。这不是勇气问题，是清晰度问题。
>
> 老子：你的焦虑来自试图同时抓住两条路。水不会这样——它选一个方向，然后全部力量都在那个方向。
>
> 毛泽东：家庭是你的基本盘，不是你的包袱。你说的"顾虑"其实是资源，看你怎么用。

*旁白：*
> "Harvest Round。
> 12 个哲学视角，同时评价你这个人。
> 不是建议——是直接说你哪里有盲区。
> 这是任何其他产品给不了的。"

**[拍摄要点]**
- 这几条评价要足够锋利、够真实，让人看完有被击中的感觉
- 如果 API 实时出不来，提前准备 mock 数据，这段用假数据录制完全可以

---

### OUTRO（1:50-2:00）— 切回黑屏字幕

**画面：黑底，白字**

```
你的大脑
第一次可以同时容纳 12 种哲学框架
```

（下方小字出现）

```
参谋 · Counsel AI
一个人的盲区，参谋来照亮。
```

*旁白：*
> "你的大脑，第一次可以同时容纳 12 种哲学框架。"
>
> （停顿 2s）
>
> "参谋。"

## 后期剪辑说明

| 片段 | 时长 | 来源 | 处理 |
|------|------|------|------|
| INTRO 字幕 | 0:00-0:25 | CapCut/Final Cut 制作 | 黑底白字逐字出现 |
| 产品界面展示 | 0:25-1:50 | QuickTime 屏幕录制 | 原始录制，轻微加速或保速 |
| OUTRO 字幕 | 1:50-2:00 | CapCut/Final Cut 制作 | 黑底白字 |
| 旁白 | 全程 | 外录或内录 | 去底噪，轻微压缩 |
| 配乐 | 全程 | Ambient/Lo-fi（无歌词）| 音量约旁白的 15-20%，高潮时淡出 |

**配乐推荐关键词：** "dark ambient instrumental" / "midnight thinking" / 或直接用 GarageBand 里的 ambient 预设

---

## 拍摄顺序建议

1. **先拍 Shot 5（12 人并行）** — 这是最难控制的，先拍好，留最多 take
2. **拍 Shot 6（Harvest Round）** — 如果 API 不稳定用 mock 数据
3. **拍 Shot 3-4（输入 + Facilitator）** — 相对简单
4. **拍 Shot 2（界面展示）** — 最简单
5. **INTRO/OUTRO 在剪辑时制作**，不需要录屏

---

## 如果时间紧，最小可行版本（60s）

砍掉 Shot 4（Facilitator 定义），其余全保留，旁白精简 30%。

60s 版结构：
- 0:00-0:15 黑屏痛点
- 0:15-0:25 界面出现
- 0:25-0:35 输入困惑
- 0:35-1:00 12 人并行（最长，25s）
- 1:00-1:20 Harvest Round
- 1:20-1:30 黑屏收尾 + 口号
