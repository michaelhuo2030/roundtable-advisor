## 产品哲学

> "一个人的盲区，参谋来照亮。"

核心信念：
- 你的顾问是你人生路径的囚徒。12个从未被你污染过的大脑，才是真正的外脑。
- AI私董会不是角色扮演，是12种哲学体系真实冲突后的洗礼。
- 价值是自己长出来的——因为我们自己每天都要用它。

## 技术架构

### 基础技术选择
- **框架：** Claude Code SDK（多Agent协调）
- **存储：** 本地文件系统（markdown + JSON）
- **界面：** Web UI（不做CLI，因为交互严重依赖UI：点击幕僚头像、查看背景、进度条）
- **并行：** Phase 4用 `Promise.all()` 并行调用12个sub-agent

### 文件结构
```
counsel-ai/
  sessions/
    [project-name]/          # 每个"项目"（持续追踪的主题）
      user-wiki.md           # 案主画像（跨session累积）
      session-001/
        00-raw-input.md      # 原始困惑
        01-defined.md        # 锁定问题 + 幕僚列表
        02-facts/            # 挖事实记录（每个幕僚一个文件）
        03-opinions/         # 12幕僚独立意见（并行生成）
        04-dimensions.md     # 冲突维度（Facilitator提炼）
        05-debate.md         # 辩论记录
        06-summary.md        # 总汇总
        07-harvest.md        # 摘果子 + to-do
        session-meta.json    # 状态、时间戳、参数
      session-002/
        context-from-prev.md # 自动注入：上轮summary
        ...
  personas/
    12-personas.md           # 12幕僚定义
  prompts/
    facilitator-agent.md
    secretary-agent.md
    hypothesis-translator-agent.md
    persona-agent-framework.md
```

## 完整用户旅程（8步）

### 顶部进度条（始终可见）
```
[✓ 输入] → [✓ 定义] → [→ 挖事实] → [ 发言] → [ 拆维度] → [ 辩论] → [ 汇总] → [ 摘果子]
```
每个milestone点击可跳转查看历史内容，不可跳过未完成步骤。

---

### Step 0：首页 · 项目列表

**展示：**
- 用户所有project卡片（项目名 + 上次session时间 + 进度）
- 每个project可以：
  - 点击 → 查看历史（只读）
  - 「继续」→ 基于上次context开启新session
  - 「新话题」→ 同一项目下开启全新session（幕僚设置可复用）

**创建：**
- 右上角「+ 新项目」→ 进入Step 1

---

### Step 1：输入困惑

**UI：**
- 大文本框，提示语："你现在最想解决的是什么？不用完美，直接说。"
- 字数不限，可以很乱，很长

**后台：**
- 保存到 `00-raw-input.md`
- 进入Step 2

---

### Step 2：定义问题（Facilitator Agent）

**核心：** Facilitator通过对话把模糊困惑转化为可讨论的形式

**流程：**
- Facilitator提问 → 案主回答 → 重复（有机轮数，由Facilitator判断够了没有）
- 案主可以说"就这样了"提前锁定
- 案主可以说"再问一个"延伸

**输出（锁定后）：**
```
## 锁定问题（1-3个）
1. [可讨论的问题表述]
2. [...]

## 推荐幕僚阵容（12人）
- 乔布斯：[理由]
- 老子：[理由]
...

## 假设检验模式 / 哲学探索模式
[Hypothesis Translator Agent判断]
```

**案主可以：**
- 修改幕僚列表（增删，但默认12人）
- 确认后进入Step 3

---

### Step 3：挖事实（简化版）⚠️ MVP保留但简化

**原则：** 每个幕僚可以向案主提一个最重要的问题；如果前面的对话已经回答了，幕僚可以主动跳过。

**简化方案（MVP）：**
- 不做串行对话，改为：**每个幕僚生成0-1个问题**（并行生成）
- UI展示：12个问题卡片（有跳过的显示"无需追问"）
- 案主统一回答所有问题（一个大文本框，按问题编号回答）
- 不支持幕僚追问（MVP）

**UI：**
```
[乔布斯] 在你描述的这个决策里，你最害怕放弃的是什么？
[老子]   （无需追问）
[保罗格雷厄姆] 你有没有找过真正用过你这类产品的人谈过？
...

[案主回答区]
1. 乔布斯：...
3. 保罗格雷厄姆：...
[提交]
```

**后台：**
- 12个问题保存到 `02-facts/questions.md`
- 案主回答保存到 `02-facts/answers.md`
- 进入Step 4

---

### Step 4：独立发言（并行 Sub-agents）⚠️ 最强demo时刻

**核心：** 12个幕僚基于完整前置上下文（步骤1-3）并行生成意见，互不可见

**Context注入（每个sub-agent）：**
```
[原始困惑] + [锁定问题] + [案主事实回答] + [该幕僚的persona prompt]
注意：不包含其他幕僚的内容
```

**UI：**
- 12张卡片同时出现，每张streaming流出内容
- 每张卡片：头像 + 名字 + 简介一行 + 意见正文（折叠/展开）
- 点击卡片头像 → 展开侧边栏：幕僚详细背景 + 哲学框架 + 历史观点

**输出格式（每个幕僚）：**
```
核心立场（1句话）
深度分析（200-400字）
我最担心的风险
我认为被忽视的角度
给案主的一个问题（留给Step 5/6用）
```

**后台：**
- 保存到 `03-opinions/persona-[name].md`
- 进入Step 5

---

### Step 5：拆维度（Facilitator提炼冲突）

**核心：** Facilitator读取12份意见，提炼3-6个冲突维度

**输出：**
```
## 冲突维度

### 维度1：短期存活 vs 长期愿景
- 正方阵营（Jobs, PG, Musk）：...
- 反方阵营（老子, 六祖）：...
- 核心张力：...

### 维度2：...
```

**UI：**
- 每个维度一张卡片，显示支持/反对阵营
- 案主多选：「选择你想深入讨论的维度」
- 最多选3个（MVP限制）
- 「全选」快捷按钮

---

### Step 6：辩论

**MVP版本：** 每个维度跑Round 1（正反中各自发言）

**Round 1：**
- 所有幕僚就该维度发言（约100-150字）
- 分组展示：正方 / 反方 / 中间派
- Facilitator提炼Round 1核心冲突（一段话）

**Next Todo（不在MVP）：**
- Round 2：各方回应对方
- 无限用户追问

**Facilitator收尾：**
- 询问案主："这个维度到此为止？还是继续？"（两个按钮）
- 下一个维度，直到所有选中维度跑完

---

### Step 7：汇总（Secretary + Facilitator）

**总汇总（自动生成）：**
```
## 总汇总

### 达成共识
- ...

### 核心分歧（未解决）
- ...

### 假设验证情况
- 得到验证：...
- 被推翻：...
- 仍存疑：...

### 关键洞见（跨幕僚）
- ...
```

**幕僚单独汇总（按需触发，MVP跳过）：**
- 点击某幕僚头像 → 触发"生成TA的完整观点总结"
- 不点则不生成（省token）

---

### Step 8：摘果子（Harvest Round）

**A：幕僚评价案主（Part B，真刀真枪）**
- 取3-5个最有代表性的幕僚
- 每人：优点（1条）+ 盲区（1条）+ 一句话建议

**B：案主自我收获**
- 文本框：「这一轮你最大的收获是什么？」
- 案主输入后保存到 user-wiki.md（自动追加）

**C：To-do提取**
- Secretary从全程对话中提取承诺/行动项
- 输出checklist

## Session-to-Session 自动上下文继承

当用户在同一项目下开启新session时，**自动注入：**

```markdown
## 上一轮背景（Session [N]，[日期]）

### 当时讨论的问题
[从01-defined.md读取]

### 关键结论
[从06-summary.md读取]

### 未解决的核心分歧
[从06-summary.md读取]

### 上轮to-do执行情况
[从07-harvest.md读取，本轮开始时让案主自评]
```

Facilitator在Step 2会先询问上轮to-do执行情况，再进入新问题定义。

## Demo模式 ⚡

**目的：** 快速跑完全流程用于hackathon现场演示

**实现：**
- 一个「Demo」按钮，点击后进入demo模式
- 预设一个虚拟案主（"Michael"）+ 预设困惑："我要不要离开现在的公司去全职创业？"
- 每个步骤案主的回答由「Fake Michael」agent自动生成（基于CLAUDE.md中的Michael画像）
- 速度模式：各幕僚意见限制在100字以内
- 完整跑完8步，约3-5分钟

**Demo的真实性：** 即使是demo模式，12幕僚依然真实调用API，体现真正的多智能体并行能力

## 关键研究支撑

本产品设计基于以下学术/实践支撑：

- **Du et al. (2023)** — 多agent辩论：价值来自初始多样性，不是互相说服
- **NGT（名义群体技术）** — 先独立生成，再交叉评估，避免groupthink
- **Delphi法** — 多轮迭代，传递完整推理过程（非摘要）
- **双钻石模型** — Phase 1：定义对的问题；Phase 2-5：找对的答案
- **Pre-Mortem** — 假设已经失败，识别30%更多风险
- **冲突提取** — Facilitator找3-6个维度，全员就每个维度辩论

## Future Plans（MVP后迭代路线图）

> 以下功能已完整设计，MVP阶段暂不实现，但架构上预留接口

### v1.1 — 辩论深度
- **辩论Round 2：** 各方基于Round 1回应对方，真正的交锋
- **用户无限追问：** 案主可在辩论中插入问题，所有幕僚回应；Facilitator定期提醒是否结束该维度

### v1.2 — 幕僚深度
- **幕僚单独汇总（点击触发）：** 点击幕僚头像 → 生成TA在本session所有发言的完整观点总结（省token，按需触发）
- **挖事实深度版：** 从简化版（一轮问答）升级为真正串行对话——幕僚看到案主回答后可追问，直到该幕僚满意

### v1.3 — 上下文智能
- **User Wiki自动深化：** 从stub升级为结构化AI提炼——每轮结束后自动分析新增内容，判断是否更新三观/决策模式/事实档案，避免重复记录
- **Session-to-session智能比较：** 不只是注入上轮summary，还自动对比：这轮的立场和上轮有什么变化？哪些假设被新事实更新了？

### v1.4 — 协作与分享
- **多人私董会：** 多个真实用户各自扮演一个幕僚角色，与AI幕僚混合
- **幕僚Marketplace：** 用户可以定制自己的幕僚并发布，他人可订阅（"你的思维可以被订阅"）
- **语音交互：** 整个流程支持语音输入/输出，变成真正的"私董会会议"体验

### v1.5 — 智慧引擎升级
- **Wisdom Engine Level 2-4：** 从persona prompt → RAG知识库 → 行为模型 → 完整智慧还原（见persona-agent-framework.md）
- **真实历史文本训练：** 每个幕僚基于真实著作/访谈/决策记录微调，而不只是系统提示词
