## 完整流程（文字版 Master Reference）

```
⚙️ 会前准备
  用户说出困惑
    → Hypothesis Translator：困惑 → 可验证假设（或广泛哲学探讨模式）
    → Facilitator：预估今日时长
                   默认召集全部12位幕僚
                   [仅以下情况减少人数]：话题简单 / 案主明确指定只要某几位
                   指定Red Team 2-3人
    → Secretary Agent：初始化档案 · 启动Checklist

◇ 双钻石 菱形1 — 搞对问题

  Phase 1 — 问题界定（发散→收敛 ~15%）
    案主陈述困惑
    → 幕僚轮流提澄清问题（仅问，不建议，不评判）
    → Facilitator重构："你真正在决定的是..."
    → 案主确认 or 修正
    → 锁定真实问题 ✓

◇◇ 双钻石 菱形2 — 搞对答案（两轮）

  Phase 2 — 独立建仓（发散 ~15%）
    全体幕僚独立生成观点（NGT模式：互不可见，无社交压力）
    → Secretary记录全部观点
    → Double-check：幕僚是否真正回应了锁定的问题？

  Phase 3 — 冲突提炼 ⭐ KEY STEP
    Facilitator读取全部观点
    → 识别3-6个不同维度的冲突（不调和之处）
    → 每个冲突单独列出，例如：
        冲突维度1：关于时间优先级的根本分歧
        冲突维度2：关于是否信任外部资源的分歧
        冲突维度3：关于短期生存 vs 长期愿景的分歧
        ...

  Phase 4 — 逐冲突深度探讨（~35%）

    对每个冲突维度，全体（或大多数）幕僚共同参与：
      → 幕僚在该冲突上自然分化为：
          正方（支持某立场）
          反方（反对该立场）
          调和者（寻求更深层balance）
      → 每个冲突内部充分探讨，不草草了事
      → Red Team全程监控，挑战所有冲突中的新兴共识

    [卡住/循环] → TRIZ：我们在这个冲突上必须停止哪个假设？
    [进展中]    → 进入下一个冲突维度
    [全部探完]  → Groan Zone：Facilitator命名仍未解决的核心张力

    ── 案主主动提问选项（新增）──
    案主可在任何冲突探讨结束后，提出新问题交给全体幕僚处理
    这是案主的主动选项，不强制
    可多轮：案主提问 → 幕僚回应 → 案主追问 → 继续

  Phase 5 — 综合与收获（收敛 ~20%）

    [双层输出]
    表层：共识摘要 · 各冲突维度结论 · 未解决张力 · 立场分布
    深层：点击各幕僚 → 完整推理逻辑 · 可转入1对1私聊模式

    [贝叶斯更新]
    Prior → Evidence（来自辩论）→ Posterior
    假设：已验证 / 证伪 / 待定

    ─────────────────────────────────────
    心理准备（Psychological Frame）
    Facilitator声明：
      "接下来你会听到真实的、锐利的评价。
       这不是攻击，是因为我们真的花时间思考了你。
       真正有价值的东西都是sharp的——不要假模假式。
       请以开放的心接收。"
    ─────────────────────────────────────

    [收获轮 Harvest Round]

    Part A — 幕僚自我收获
      每位幕僚：
        · 我今天的收获是什么？
        · 我从其他幕僚那里学到了什么？

    Part B — 幕僚评价案主 ⭐（极珍贵）
      每位幕僚：
        · 案主今天的表现如何？
        · 我对他有了什么新认知？
        · 我在他身上看到了什么？（优点 / 盲点 / 未被看见的东西）
      真刀真枪，不要你侬我侬

    Part C — 案主自我收获
      案主：
        · 我今天最重要的收获是什么？
        · 我听到了什么让我意外的？

    Part D — Facilitator收尾
      提炼集体收获
      确认案主的收获感
      标注本次session的"最珍贵时刻"

🗂️ 归档 — Secretary
  归档完整Session记录
  To-Do清单 · 关键洞察 · 收获摘要 · 破防时刻标注（若有）
  → To-Do导出 → Claude Code本地知识库 · 个人Journal

🔄 真实飞轮（跨Session）
  用户现实执行To-Do
  → 回来复盘：预期 vs 实际发生了什么？
  → 一键导入上次Context，已知信息自动跳过，幕僚提问更精准
  → 新一轮私董会：Context累积，效率倍增
  → 无限循环 ↩
```

## 角色定义

| 角色 | 类型 | 核心职责 |
|------|------|---------|
| Hypothesis Translator | Agent | 将困惑翻译为可验证假设 |
| Facilitator | Agent（仅流程，不发内容观点） | 预估时长 · 召集幕僚 · 重构问题 · 提炼冲突维度 · 心理frame · 命名张力 |
| Secretary | Agent | 记录 · 归档 · Checklist · double-check |
| 幕僚 Personas | Agents × 12（默认全员） | 内容贡献者，各有独立哲学框架，在每个冲突上自然分化 |
| Red Team | Agents × 2-3 | 全程挑战共识，轮换担任 |

**幕僚人数规则：**
- 默认：全部12人
- 减少至6-8：话题简单 / 案主明确指定
- 原则：复杂问题尽量全员，多元视角是核心价值

## 冲突探讨机制（修正说明）

**错误理解（旧版）：** 把冲突按性质分成"正反辩论组"和"平衡组"，各组只管自己的冲突

**正确理解（v4）：**
- Facilitator识别3-6个冲突**维度**（如：时间优先级 / 资源信任 / 短长期权衡）
- **每个冲突**都由全体幕僚共同参与探讨
- 在每个冲突内部，幕僚**自然分化**为正方/反方/调和者
- 每个冲突都深度挖到底，不是走过场

## 收获轮设计原则（Radical Candor）

- Care personally（真的在意你）→ 心理准备frame
- Challenge directly（所以直接说）→ Part B真刀真枪
- Part B是整个产品最独特的价值：案主同时从12个哲学视角被评价，这是任何其他产品给不了的

```
Session结束 → To-Do清单
      ↓
 用户现实执行
      ↓

      ↓
 一键导入上次Context

      ↓ （无限循环）
```

## 学术引用

> Du, Y., Li, S., Torralba, A., Tenenbaum, J. B., & Mordatch, I. (2023).
> **Improving Factuality and Reasoning in Language Models through Multiagent Debate.**
> *arXiv preprint arXiv:2305.14325.*
> https://arxiv.org/abs/2305.14325

核心发现：多智能体辩论价值来自**初始观点多样性**，非相互说服。
产品定位：让用户同时看到多种哲学框架怎么解读同一问题，而不是找"正确答案"。
