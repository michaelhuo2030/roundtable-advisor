## Quality Checklist for Coder

### Before Starting Each Profile:
- [ ] Review Explorer's specific task list for this master
- [ ] Identify primary source materials
- [ ] Plan section structure before writing

### During Supplementation:
- [ ] **Quotes:** Verify each quote with source citation
- [ ] **Biography:** Use timeline format with significance notes
- [ ] **Philosophy:** Explain concepts with concrete examples
- [ ] **Case Studies:** Follow Problem → Context → Analysis → Solution → Execution → Result → Principle format
- [ ] **Persona:** Include specific speech patterns, catchphrases, rhetorical devices
- [ ] **Sources:** Prefer primary sources (books, speeches, interviews) over secondary

### Before Marking Complete:
- [ ] File size meets minimum (7KB for B-level, 15KB for A-level)
- [ ] Line count meets minimum (250 for B-level, 400 for A-level)
- [ ] Quote count meets minimum (15 for B-level, 20 for A-level)
- [ ] Case study count meets minimum (2 for B-level, 3 for A-level)
- [ ] All sections present and substantial (not bullet points only)
- [ ] Sources section updated with citations
- [ ] Cross-reference with Laozi profile for format consistency
