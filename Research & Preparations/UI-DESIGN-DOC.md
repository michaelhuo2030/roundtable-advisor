# 参谋 · UI Design Document
> 记录所有设计决策的来龙去脉，以便团队日后迭代

---

## 产品简介

**参谋**是一个 AI 驱动的"十二人智库"系统：用户描述困境，12位历史上最强的战略思维者（乔布斯、毛泽东、老子、六祖慧能、钱学森、爱因斯坦、Kevin Kelly、Paul Graham、Jeff Bezos、Elon Musk、Bruce Lee、a16z）以各自独特的视角发言、辩论，帮助用户照亮盲区、作出更好的决策。

**核心定位：** 不是聊天机器人，是私人智库圆桌会议。你是发起人，他们是幕僚。

## 设计进化史

### v1 — 原型验证
- 基础 Ring（星环）和 Table（圆桌）双布局
- 绿色配色（`#00F3A8`）
- 文字放底部，遮挡视觉
- 只展示 3 名幕僚（Jobs/PG/Mao）
- **问题：** APP感太强，没有沉浸感

### v2 — 布局探索
- 验证 Ring (A) + Isometric Table (B) 双视角切换
- 引入右侧文字面板（约1/4宽度）
- 用 JS 三角函数计算 12 节点位置：`x = cx + r·sin(deg)`, `y = cy - r·cos(deg)`
- **进步：** 文字不再遮挡场景

### v3 — 游戏化探索
- 完全去掉 APP 导航，改用悬浮触发的隐藏导航栏（hover 显现）
- 角色轮廓出现在场景中
- **问题：** 颜色（绿色）无论怎么调都显得"土"

### v4 — 琥珀金 + 布局确定
- 换色：绿 → 琥珀金 `#C9A96E`
- 确定最终布局：场景永远在左，文字永远在右（不再切换）
- 引入 12 名幕僚全部在场
- **进步：** 有高级感，但金色仍过重

### v5 — 纯黑白 + 全员在场
- 换色：琥珀金 → 纯黑白（Jobs 审美：`#fff` / `rgba(255,255,255, x)`）
- 多层白色透明度创造深度感（0.06 / 0.18 / 0.28 / 0.55）
- 三条同心轨道环 `.orb` 增加宇宙感
- 左下角加入 📝记录员 + ⚖️调解员（球童风格）
- 圆桌表面可见：多层 inset shadow + 内边框
- 实现：点击角色 = 打开档案；再次点击 = 关闭（`curOpenChar` 追踪）
- **问题：** 
  - 左下角有一个小方块（`.staff-desk`）让人困惑
  - 档案是独立的 overlay panel，不够系统化
  - 步骤3所有问题同时显示，缺乏交互节奏感

### v6 — 模式系统 + 顺序提问
**核心架构升级：右面板多模式系统**
- 移除 `.staff-desk`（困惑方块）
- 移除独立档案 overlay，档案并入右面板模式
- 5种模式：`step`（步骤）/ `profile`（档案）/ `speech`（发言）/ `wiki`（记录）/ `status`（状态）
- 步骤3改为顺序提问（一人一问，场景中高亮当前提问者）
- 步骤2改为多选勾选 + 自定义输入
- 气泡可点击（`pointer-events:auto`），点击展示完整发言
- 维度追踪器（从步骤5起显示在场景右上角）
- 幕僚档案增加：哲学体系、方法论、价值观 pills、关键人生事件

### 右面板模式系统
| 模式 | 触发方式 | 显示内容 |
|------|---------|---------|
| `step` | 默认/返回 | 当前步骤的交互内容 |
| `profile` | 点击角色头像 | 该幕僚的完整档案 |
| `speech` | 点击气泡 | 该幕僚的完整发言（markdown格式） |
| `wiki` | 点击 📝 记录员 | 本次 Session 的完整记录 |
| `status` | 点击 ⚖️ 调解员 | 立场温度计 + 调解建议 |
| `lobby` | 初始/大厅 | 游戏式大厅（Session历史、档案库、Wiki入口）|

**后退逻辑：**
- 在 Session 中 → 后退返回 `step` 模式
- 在大厅中 → 后退返回 `lobby` 模式

### 场景图层层级

### 角色状态系统
每个角色在每个步骤有明确的视觉状态：

| 状态 | 视觉表现 | 含义 |
|------|---------|------|
| `dim` | opacity 0.15 | 未激活，等待 |
| `active` | 白色边框微发光 | 参与中，关注 |
| `speaking` | 白色脉冲动画 + ripple | 正在发言 |
| `done` | opacity 0.6 | 已发言完毕 |
| `waiting` | opacity 0.1 | 极度弱化（辩论时非参与者）|
| `pro` | 强白色边框 + glow | 辩论正方 |
| `con` | 弱边框 + 半透明 | 辩论反方 |

**辩论步骤的视觉设计（重要）：**
步骤6辩论时，非辩论者状态设为 `waiting`（opacity:0.1），辩论双方以 `pro`/`con` 高亮对比。这产生了极强的戏剧效果：黑暗中只有辩论者发光。

### 步骤3：顺序提问机制
`S3_QUEUE` 定义了提问顺序（5人）：
1. 乔布斯 → "idea本身 vs 创业者身份认同？"
2. Paul Graham → "找过10个潜在付费用户吗？"
3. 毛泽东 → "合伙人能力互补还是重叠？"
4. Jeff Bezos → "10年后怎么看今天这个决定？"
5. 老子 → 沉默（skip:true，自动显示"老子此刻选择沉默"）

每人提问时：
- 场景中该角色高亮为 `speaking` 状态
- 右面板显示该角色的问题 + 回答输入框
- "完整发言 ↗" 按钮打开 speech 模式
- 用户可以跳过或提交回答
- 已回答的问题折叠到下方列表

### Ring 布局坐标计算
```js
// 半径210px，在540×540的容器中心
const r = 210, cx = 270, cy = 270;
for (let i = 0; i < 12; i++) {
  const rad = i * 30 * Math.PI / 180; // 12等分，30°一个
  const x = cx + r * Math.sin(rad);   // 12点钟位置是 i=0
  const y = cy - r * Math.cos(rad);
}
```

### 气泡设计
- 静态气泡：简短摘要，有 `speechId` 的可点击（`pointer-events:auto`）
- 点击气泡 → 右面板显示该幕僚的完整发言（speech 模式）
- 悬停提示："点击查看完整发言 ↗"
- 步骤3气泡：由 `renderS3Step()` 动态注入，随当前提问者更新

## 8步会议流程

| 步 | 名称 | 核心交互 | 场景状态 |
|---|------|---------|---------|
| 1 | 说出你的困境 | 大文本输入框 | 全暗 |
| 2 | 主持人定义问题 | 对话 + 多选框架 | 主持人发言，12人active |
| 3 | 幕僚在问你 | 顺序提问，逐一回答 | 单人speaking，已答者done |
| 4 | 12位并行发言 | 气泡 + 可点击完整发言 | 部分speaking，部分waiting |
| 5 | 拆维度 | 多选维度卡片 | 主持人发言，维度追踪器出现 |
| 6 | 辩论 | 正反方对决，可插话 | pro/con高亮，其余极暗 |
| 7 | 汇总洞察 | 主持人气泡（可点击完整） | 主持人发言，全员done |
| 8 | 摘果子 | 勾选行动项 | 全员done |

## 工作人员（左下角）
- **📝 记录员**：点击 → wiki 模式，显示 Session 完整记录（按步骤组织）
- **调解员 ⚖️**：点击 → status 模式，显示幕僚立场温度计 + 调解建议
- 设计风格："球童"风格，在黑暗中低调存在，按需使用
- **没有** `.staff-desk` 矩形（v6已移除，因为令人困惑）

---

## 导航系统
- 顶部导航栏：**鼠标悬停到屏幕顶部**才显现（hover-reveal）
- 平时完全隐藏（`opacity:0; translateY(-100%)`），保持场景沉浸感
- 激活方式：`#nav-zone`（8px高触发区）hover 或 nav 本身 hover
- 导航不占任何常驻界面空间

## 颜色系统
**纯黑白方案（最终确定）**

| 变量 / 用途 | 值 |
|------------|---|
| 背景 | `#000000` |
| 主色文字 | `#ffffff` |
| 强调文字 | `rgba(255,255,255,0.65~0.8)` |
| 次级文字 | `rgba(255,255,255,0.4~0.5)` |
| 弱文字/标签 | `rgba(255,255,255,0.2~0.3)` |
| 背景层次1 | `rgba(255,255,255,0.03~0.05)` |
| 背景层次2 | `rgba(255,255,255,0.07~0.1)` |
| 边框弱 | `rgba(255,255,255,0.07~0.12)` |
| 边框中 | `rgba(255,255,255,0.18~0.25)` |
| 边框强 | `rgba(255,255,255,0.35~0.5)` |
| 轨道环主 | `rgba(255,255,255,0.22)` |
| 轨道环次 | `rgba(255,255,255,0.08)` |

**为什么纯黑白？** 任何有色系（绿、金、蓝）在这个"极度严肃的高端智库"场景中都显土。黑白的对比度、纯粹性本身就是一种立场。

## 技术实现关键点

**单文件 HTML**：所有代码（HTML/CSS/JS）在一个文件中，无依赖，无构建，浏览器直开。

**关键JS函数：**
- `setMode(mode, data)` — 右面板模式切换核心
- `show(stepId)` — 步骤切换 + 场景/气泡/角色同步更新
- `renderS3Step()` — 步骤3顺序问询渲染
- `updateChars(stepId)` — 更新所有角色的视觉状态
- `updateBubbles(stepId)` — 动态注入气泡（点击事件 + speechId绑定）
- `updateDimTracker(stepId)` — 显隐维度追踪器
- `populateProfile(id)` / `populateSpeech(id)` — 右面板内容填充

**无用状态保护：** 步骤3气泡由 `renderS3Step()` 专管，不经过通用 `updateBubbles()`（后者对 s3 直接返回）。
